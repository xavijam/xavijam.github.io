CartoDB

Create beautiful dynamic data driven maps

2012

CartoDB is a geospatial database on the cloud that allows for the storage and visualization of data on the web. User could create dynamic maps, analyze and build location aware and geospatial applications with your data using the power of PostGIS in the cloud.

It was released as a Beta version at Denver FOSS4G in September 2011, and nowadays we are working hard on the second version of the application.

The tool uses JavaScript extensively, including use in the front-end web application and back-end Node.js based APIs for the client libraries.

We have built the tool on a stack of some of our favorite open-source technologies, including, PostgreSQL, PostGIS, Mapnik, and our very own WindShaft. CartoDB is no different, you can find the code in our git repository.

I've spent last years working on the client-side, UI and front-end develpoment, based mainly on Backbone.js library.

Behind the scenes

Some open frameworks, plugins or libraries I’ve used working on this project

  • CartoDB.js

    JS library for creating CartoDB maps.
  • Backbone.js

    JS framework for structuring JS as MVC.
  • jQuery

    JS library for manipulating HTML.
  • LeafletJS

    Open-source JS map framework.
  • Modernizr

    JS library that detects HTML5&CSS3 features.
  • jQuery UI

    Set of JS UI interactions, effects, widgets...
  • Codemirror

    Text editor implemented in JS.
  • D3.js

    JS library for handling docs based on data.
  • Select2

    JS replacement for select boxes.
  • Compass

    Open-source CSS Authoring Framework.
  • Spin.js

    Creates a JS spinning activity indicator.
  • Tag-it

    Tag editing JS widget.
